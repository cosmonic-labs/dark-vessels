Copernicus SAR Data

Sentinel-1 provides free, open-access Synthetic Aperture Radar (SAR) imagery covering the world's oceans. SAR can detect ships regardless of weather or time of day.

To connect to live satellite data:

  1. Create a free account at dataspace.copernicus.eu
  2. Go to your Dashboard and generate an API token
  3. Click "Connect" on the data sources panel and enter your credentials

Without credentials, the system uses synthetic SAR data for demonstration.

Connect to Copernicus CDSE

Enter your Copernicus Data Space credentials. These are stored locally in your browser and sent only to the Copernicus API.

Your credentials are stored in a browser cookie and never leave your machine except to authenticate with Copernicus.

AISHub Vessel Telemetry

AIS (Automatic Identification System) is a maritime tracking system. Ships broadcast their identity, position, and course via radio. AISHub aggregates this data from a global network of receivers.

To get live AIS data:

  1. Register at aishub.net
  2. To receive an API key, you need to share an AIS receiver or purchase a data plan
  3. Once approved, your API key appears in your account dashboard

Without a key, the system uses synthetic AIS data with realistic vessel names and positions.

Enter AISHub API Key

Paste your AISHub API key below. This is stored locally in your browser and sent only to the AISHub API.

Your key is stored in a browser cookie and never leaves your machine except to query AISHub.

GPU Compute Pipeline

This system uses server-side WebGPU via the WASI WebGPU interface to run signal processing on the GPU. Here's how the pipeline works:

  1. SAR Image Input — A Sentinel-1 radar intensity image (or synthetic equivalent) is loaded as a 2D array of floating-point values
  2. GPU Buffer Upload — The image data is written to GPU storage buffers via wasi:webgpu
  3. WGSL Compute Shader — A CA-CFAR (Cell-Averaging Constant False Alarm Rate) shader runs on the GPU with 16×16 workgroups. Each thread evaluates one pixel against its surrounding clutter level
  4. Result Readback — Detection mask is copied back from GPU to CPU memory
  5. Post-processing — Connected component labeling groups detected pixels into ship objects with estimated dimensions

When is the GPU used?

The GPU processes the CFAR algorithm on both synthetic and real SAR data — the compute shader operates on the pixel intensity array regardless of source. With real Sentinel-1 GRD images (25,000+ × 16,000+ pixels), GPU acceleration becomes essential. A 512×512 demo image processes in ~300ms on CPU; a full satellite scene would be 2,500× larger.

The GPU path activates when the wasmCloud host is started with --wasi-webgpu. Without it, processing falls back to CPU automatically.
